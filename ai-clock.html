    About

    In 2026 I was working as a sub teacher. Sometimes it was convenient to have a timer or a clock for my classes and I used ones I found online. But, suddenly the clock had lots of intrusive ads put in it which needed to be closed before being used. After a quick search I couldn't find anything exactly what I wanted, so I decided to write my own.

    I did - it is clock.html. It does everything I need and a bunch more: there are always bells and whistles that can be added. I spent a week of free time on it. On finishing I thought it might be a good chance to test out AI for coding, so I went to copilot and started.

    In all I spent under 2 hours on it. It does not have all the features mine does, but it does have all the important ones, and another half hour would probably give it the rest. The code is simpler - very straightforward, and compared to clock, which uses objects to provide a more elegant solution. If someone wants to fix or modify the code the AI code has the advantage that it is probably simpler to understand at first glance - to modify OO code you need to understand the objects, which might take more time. On the other hand, for more complicated changes, enhancement, or code reuse the OO code is better.

    I was also surprised at some of the mistakes in the code. Several times I'd put in the changes that the AI sent me and found errors that were similar to what a careless programmer (like me) might make - leaving in references to replaced variable names, putting listeners in the wrong place, etc. I did not expect this type of error. The CSS organization was much cleaner and better than mine.

    In a project this small it doesn't make much difference, but in a larger one the straightforward approach will get messy. The tldr I get from this is that for writing shorter modules AI is a wonderful time saver, it is probably best to have a design in mind and use AI to fill in the modules, much like a junior programmer or a consultant might do.
